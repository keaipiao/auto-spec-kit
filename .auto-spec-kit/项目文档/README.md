# 项目文档目录说明

本目录存放当前项目长期有效的规格化文档。

任务档案记录某一次需求、迭代、Bug、重构或调研的过程；项目文档记录经过确认后仍然对当前项目有效的事实。

## 建议文档

- 项目总览.md
- 产品定位.md
- 当前需求规格.md
- 产品设计.md
- UIUX设计规范.md
- 技术选型.md
- 架构设计.md
- 数据模型.md
- 接口契约.md
- 测试策略.md
- 验收标准.md
- 发布策略.md
- 决策记录/

## 回写规则

任务完成后，AI 应判断哪些结论需要从任务档案回写到项目文档。

例如：

- 功能迭代改变了当前需求，应更新 `当前需求规格.md`。
- UI/UX 方向改变，应更新 `UIUX设计规范.md`。
- 技术方案改变，应更新 `技术选型.md` 或 `架构设计.md`。
- 重大决策应写入 `决策记录/`。

不要把未确认的临时过程记录写入项目文档。
