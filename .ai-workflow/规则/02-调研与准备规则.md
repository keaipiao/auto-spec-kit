# 调研与准备规则

## 调研不只是竞品调研

调研包括所有让后续规格和开发不被卡住的前置工作。

常见调研类型：

- 产品调研。
- 市场调研。
- 用户调研。
- 竞品调研。
- UI/UX 调研。
- 技术调研。
- 技术栈调研。
- 依赖和插件调研。
- API、账号、Key、权限调研。
- 部署和运行环境调研。
- 法务、隐私、成本、合规调研。

## 前期准备清单

进入开发运行合约前，必须检查：

| 类别 | 需要确认的问题 |
| --- | --- |
| 账号与权限 | 是否需要第三方账号、API Key、云服务权限、数据库权限 |
| 模型与服务 | 是否需要 LLM、Embedding、语音、图像、支付、短信等服务 |
| 本地环境 | Node、Python、Docker、数据库、浏览器、测试工具是否可用 |
| AI 技能和插件 | 是否需要安装特定 Codex skill、MCP、浏览器插件、部署插件 |
| 数据来源 | 是否有样例数据、测试数据、知识库、文件、接口 |
| UI/UX 参考 | 是否需要查设计文章、竞品截图、组件库、风格参考 |
| 测试能力 | 是否能跑单测、集成测试、E2E、截图测试 |
| 部署能力 | 是否需要 Vercel、Docker、云服务器、域名、证书 |

## 什么时候必须调研

以下情况默认需要调研，除非用户明确要求跳过：

- 新项目。
- 高复杂功能。
- AI 相关功能。
- 支付、登录、权限、隐私、数据迁移。
- 用户体验强相关页面。
- 用户不确定技术栈。
- 需要新增外部依赖。
- 需要连接第三方服务。

## 什么时候先问用户

AI 不能擅自决定以下内容：

- 产品定位。
- 目标用户。
- 商业模式。
- UI/UX 主题风格。
- 是否做竞品分析。
- 是否接入付费服务。
- 是否更换技术栈。
- 是否安装重依赖或插件。

AI 可以提出建议，但必须让用户确认。

## 调研产物

调研产物不一定都要很长，但必须能回答：

- 查了什么。
- 为什么查。
- 得到了什么结论。
- 对需求、设计、技术或开发有什么影响。
- 哪些内容还需要用户确认。
