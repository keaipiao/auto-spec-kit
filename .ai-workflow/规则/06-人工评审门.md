# 人工评审门

## 什么是关键文档

关键文档是会影响产品方向、开发范围、技术路线、成本、风险或验收结果的文档。

关键文档必须交给用户或负责人审查。

## 关键文档

| 文档 | 为什么关键 |
| --- | --- |
| 原始需求确认 | 确认 AI 没理解错用户目标 |
| 调研结论 | 决定是否值得做、怎么做 |
| 需求规格 | 决定做什么、不做什么 |
| 产品设计 | 决定流程、功能、用户体验 |
| UI/UX 方向 | 主观性强，不能由 AI 擅自决定 |
| 技术选型 | 影响长期维护、成本和风险 |
| 架构设计 | 影响系统边界和扩展性 |
| 测试与验收标准 | 决定如何判断完成 |
| 开发运行合约 | 决定 AI 可以自动开发到哪里 |
| 发布与回滚方案 | 涉及上线风险 |

## 非关键文档

非关键文档一般不需要用户逐项审查，可以由 AI 自动生成和维护。

例如：

- 开发日志。
- 中间执行记录。
- 普通测试日志摘要。
- 自动格式化结果。
- 非关键实现说明。
- 临时草稿。

但如果非关键文档暴露出风险，应升级为关键事项。

## 评审原则

- 人类审查决策，不审查每个过程细节。
- AI 在进入 GOAL 开发前必须完成关键文档确认。
- GOAL 开发中不频繁打断用户。
- 最终交付时，人类审查结果、风险、卡点和验收项。

## 用户不确定时怎么办

如果用户自己也不确定，例如是否需要竞品分析、是否需要某种 UI 风格、是否用某个技术栈，AI 不能直接替用户决定。

AI 应该：

1. 说明可选方案。
2. 说明每个方案的影响。
3. 给出推荐。
4. 等用户确认。

只有进入已授权的 GOAL 开发流程后，AI 才可以在授权范围内自行做实现级选择。
