# 技术选型

## 填写说明

本文件用于记录技术路线的选择依据。技术栈不能只写一个字段，必须说明候选方案、选择理由、不选理由、运行环境、依赖、部署和风险。

## 选型目标

- 说明技术选型要满足什么目标。
- 示例：支持快速开发单页应用、容易部署、测试成本低、适合 AI 编程工具维护。

| 方案 B |  |  |  |  |

## 推荐方案

- 写最终推荐的技术栈组合。
- 示例：React + Vite + TypeScript + Vitest + Playwright。

## 推荐理由

- 为什么选择它。
- 示例：生态成熟、启动快、测试工具完整、部署到静态托管简单。

## 不选方案及原因

- 说明主要备选方案为什么不选。
- 示例：Next.js 对纯前端 MVP 偏重；Vue 团队熟悉度不足。

## 运行环境

- Node、Python、Docker、数据库、浏览器等版本要求。

## 依赖、插件、技能、MCP

- 需要哪些依赖、AI 工具能力、MCP、浏览器能力、部署插件。
- 注意：不要只写 Codex，应按实际工具列出 Codex、Claude Code、Trae、Cursor 等是否需要配置。

## API Key、账号、权限

- 是否需要第三方账号、API Key、云服务权限。
- 如果没有，应标记是否阻断开发。

## 部署方式

- 本地运行、预览环境、生产部署方式。

## 风险和替代方案

- 技术风险、维护风险、替代方案。
