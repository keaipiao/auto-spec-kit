# 新项目流程

## 适用场景

从 0 到 1 创建一个产品、应用或独立模块。

## 流程顺序

1. 原始需求
2. 前期准备清单
3. 调研计划
4. 调研报告
5. 需求规格
6. 产品设计
7. UI/UX 设计规范
8. 技术选型
9. 架构设计
10. 数据模型
11. 接口契约
12. 测试计划
13. 验收标准
14. 开发运行合约

## 条件文档

- 有明显市场或用户不确定性时，使用 `文档模板/新项目/工程可读/调研计划.md` 和 `文档模板/新项目/工程可读/调研报告.md`。
- 有界面时，使用 `文档模板/新项目/用户可读/UIUX设计规范.md`。
- 有持久化数据时，使用 `文档模板/新项目/工程可读/数据模型.md`。
- 有 API 或外部集成时，使用 `文档模板/新项目/工程可读/接口契约.md`。

## 人工确认点

- 产品定位。
- MVP 范围。
- UI/UX 方向。
- 技术栈。
- 开发运行模式。
